====================================================
PREMIUM ARCHITECTURE FLOWCHARTS
===============================

Do NOT create large enterprise-style diagrams.

Create compact, modern, recruiter-friendly architecture visualizations that fit naturally inside each expandable case study card.

Design Goals:

* Clean and premium
* Small footprint
* Mobile responsive
* Easy to understand in under 10 seconds
* Smooth animations
* Matches existing portfolio aesthetics
* Looks similar to modern SaaS architecture cards

====================================================
UI STYLE
========

Each architecture should appear inside:

"System Architecture"

section.

Use:

* Rounded glassmorphism cards
* Gradient borders
* Hover elevation
* Animated connectors
* Subtle glow effects
* Technology badges
* Framer Motion animations
* Expand-on-scroll effects

Node Style:

[ React ]
Frontend Layer

[ Express ]
API Layer

[ MySQL ]
Database Layer

Badges:

React
Node.js
Socket.IO
MySQL
Gemini AI
TypeScript

====================================================
ANIMATION REQUIREMENTS
======================

When card expands:

* Nodes fade in sequentially
* Connectors animate drawing
* Technology badges slide in
* Architecture appears progressively

Hover:

* Slight scale effect
* Soft glow
* Border highlight

Do NOT use flashy animations.

Keep professional.

====================================================
RESPONSIVENESS
==============

Desktop:

* Flowchart centered
* Maximum width controlled
* Clean spacing

Mobile:

* Automatically stack vertically
* Maintain readability
* No horizontal scrolling

====================================================
IMPORTANT
=========

Do NOT use:

* Mermaid diagrams
* Plain text arrows
* Huge system design diagrams
* SVG diagrams requiring zoom

Use actual React UI components styled as architecture cards with animated connectors.

The architecture should feel like a premium portfolio showcase, not technical documentation.
